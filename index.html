GitFlow分支规范

本规范适用于采用 Git Flow 模式进行项目开发的团队，以确保开发流程有序、版本可控、协作高效。

分支结构

项目将使用以下主要分支：

  • master：主分支，部署到生产环境的稳定版本。
  • develop：开发分支，集成所有功能分支的代码。
  • feature/*：功能分支，从 develop 分出，用于开发新功能。
  • release/*：发布分支，从 develop 分出，用于准备发布版本。
  • hotfix/*：热修复分支，从 master 分出，用于修复生产环境的紧急问题。
  • test：测试环境分支，用于部署测试环境，通常基于 develop 或 release/*。

分支命名规范

  • 功能分支：feature/功能名称
  • 发布分支：release/版本号
  • 热修复分支：hotfix/修复描述
  • 测试分支：固定为 test

工作流程

  1. 开发新功能：从 develop 拉取 feature/* 分支进行开发。
  2. 合并功能：开发完成后合并到 develop，并删除 feature/* 分支。
  3. 准备发布：从 develop 创建 release/* 分支，修复 bug、准备文档。
  4. 发布上线：release/* 测试通过后合并到 master 和 develop。
  5. 紧急修复：从 master 创建 hotfix/*，修复完成后合并到 master 和 develop。
  6. 测试环境部署：将 develop 或 release/* 合并到 test，供测试人员验收。

标签规范

每次发布到 master 的版本应打标签，格式如下：

v1.0.0
v1.0.1

提交规范

建议使用以下格式的提交信息：

类型: 简要描述

详细描述（可选）

常用类型包括：feat、fix、docs、style、refactor、test、chore。

Java 面试题库（完整版，含标准答案）

模块一：Java 基础（共 10 题）

1. Java 的基本数据类型有哪些？

Java 有 8 种基本数据类型：byte、short、int、long、float、double、char、boolean。

2. 面向对象的三大特性是什么？

封装、继承、多态。

3. String 为什么是不可变的？

因为 String 被 final 修饰，底层使用 char 数组存储，一旦创建不可更改，提高安全性和效率（如在 HashMap 中）。

4. ArrayList 和 LinkedList 的区别？

  • ArrayList：基于数组，查找快，插入删除慢。
  • LinkedList：基于链表，插入删除快，查找慢。

5. 重载和重写的区别？

  • 重载（Overload）：同一类中方法名相同，参数不同。
  • 重写（Override）：子类重写父类方法，参数和返回值相同。

6. 抽象类和接口的区别？

  • 抽象类可以包含实现的方法，接口只能包含抽象方法（Java 8 以后允许 default 方法）。
  • 一个类只能继承一个抽象类，但可以实现多个接口。

7. Java 中的异常体系？

  • Checked Exception（受检异常）：如 IOException。
  • Unchecked Exception（运行时异常）：如 NullPointerException。

8. final、finally 和 finalize 的区别？

  • final：修饰不可变内容。
  • finally：异常处理中始终执行的代码块。
  • finalize：垃圾回收前调用的方法（不推荐使用）。

9. Java 中的包装类有哪些？

Boolean、Byte、Character、Short、Integer、Long、Float、Double。

10. Java 如何实现反射？

通过 Class.forName()、getDeclaredMethods()、getConstructor() 等方法反射获取类信息并动态操作。
